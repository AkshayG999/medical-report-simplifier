1. Introduction

MedInsight AI is a web-based health report decoder that helps users understand medical reports in simple language. Medical reports often contain technical words, values, reference ranges, abbreviations, and medical conclusions that are difficult for a normal user to understand. The purpose of this project is to upload a report file, process it using artificial intelligence, and generate a patient-friendly explanation along with recommendations and trusted health resources.

The system accepts PDF and image files such as JPG, JPEG, and PNG. It supports English, Hindi, and Marathi output, making the application useful for local users who prefer regional language explanations. The project also stores report records in a SQLite database so that completed reports can be viewed again from the frontend.

This project is developed using React and TypeScript for the frontend, Express.js for the backend API, SQLite for storage, and LangGraph with Google Gemini for AI-based report understanding.

2. System Analysis

2.1 Existing System

In the existing manual system, a patient receives a medical report from a laboratory, hospital, or diagnostic center. The user normally depends on a doctor, medical staff, or internet search to understand the meaning of test values and findings. This process is slow, confusing, and sometimes inaccurate because medical terms require proper context.

Existing online search-based methods do not understand the complete report. They only provide generic information about a single test or medical condition. Users may misunderstand results if they read unrelated or incomplete information.

2.2 Need of Computerization

Computerization is required to make report understanding faster, more organized, and more accessible. An automated system can accept medical reports digitally, extract important information, simplify complex terms, store report history, and present useful recommendations in a structured format.

The need of computerization includes quick processing, multilingual explanation, digital storage, better user experience, and easy retrieval of previously processed reports.

2.3 Proposed System

The proposed system is MedInsight AI, an AI-powered web application. The user uploads a medical report and selects a preferred language. The system saves the initial report metadata in the database, sends the file data to the AI processing graph, extracts medical information, creates a simplified summary, generates personalized insights, suggests doctor visit questions, and stores the final analysis in the database.

The application provides a clean frontend interface for uploading reports, viewing processing status, reading output, and opening saved reports. The backend exposes REST APIs for report creation, update, retrieval, deletion, and health check.

2.4 Scope and Objective of Proposed System

Scope:

  • Upload PDF and image-based medical reports.
  • Generate patient-friendly summaries from uploaded reports.
  • Provide recommendations and follow-up questions for doctors.
  • Support English, Hindi, and Marathi languages.
  • Store report records and analysis status in SQLite.
  • View previously saved reports from the frontend.

Objectives:

  • To reduce confusion caused by complex medical terminology.
  • To provide faster understanding of reports.
  • To create a simple digital workflow for report analysis.
  • To maintain a database of processed reports.
  • To improve accessibility through multilingual support.

2.5 Fact Finding Techniques

The following techniques were considered while studying the project requirements:

  • Document Study: Studied medical report formats, lab values, and common diagnostic report structures.
  • Observation: Observed how users usually read medical reports and search the internet for explanations.
  • Interview Method: Considered common user expectations such as simple language, quick result, and doctor-related questions.
  • Software Study: Analyzed modern web application architecture with frontend, backend API, and local database.

2.6 Feasibility Study

Technical Feasibility: The project is technically feasible because it uses established technologies such as React, Express.js, SQLite, LangGraph, and Gemini API.

Operational Feasibility: The system is easy to operate. Users only need to upload a report and select a language. The application displays the result in a readable format.

Economic Feasibility: The project uses open-source frontend and backend frameworks. SQLite is lightweight and does not require a separate database server for local use.

6. System Implementation

The system is implemented as a full-stack TypeScript web application. The frontend is built using React and Vite. The UI provides upload, processing, result, and saved report screens. React Dropzone is used for file selection. React Markdown is used to display AI-generated Markdown output in a readable format.

The backend is implemented using Express.js. It provides REST API endpoints under /api. SQLite is used as a local database through the better-sqlite3 package. The database is initialized automatically when the backend starts.

The AI workflow is implemented using LangGraph. It contains three main nodes: extraction, simplification, and recommendation generation. The extraction node reads the uploaded file and extracts medical facts. The simplification node converts medical content into patient-friendly language. The recommendation node generates recommendations, insights, and trusted resource links.

8. Limitations

  • The system depends on an external AI API, so internet connectivity and API availability are required.
  • The generated explanation is for informational purposes only and cannot replace professional medical advice.
  • Accuracy depends on the quality and clarity of the uploaded report file.
  • Very complex or unclear reports may produce incomplete extraction.
  • The current database is local SQLite, which is suitable for development but not ideal for large multi-user production deployment.
  • Authentication and role-based access control are not implemented in the current version.

9. Conclusion

MedInsight AI successfully demonstrates how artificial intelligence can be used to simplify medical reports for common users. The system provides report upload, multilingual summary generation, personalized recommendations, trusted resources, and saved report viewing. The combination of React, Express.js, SQLite, LangGraph, and Gemini API creates a practical full-stack solution.

The project helps reduce confusion caused by complex medical language and supports users in preparing better questions for their healthcare providers. It is not a diagnostic tool, but it is useful as an educational and assistive application.
